# Event Management
Create a nextjs project using typescript and tailwing where we will have an Event Management Panel.
The main page should have:
1. An input to filter by Event name with debounce
2. Button to order by date
3. Filter by status
3. Cards of events. Each card should have
  a. Status
  b. Name
  c. Data
  d. Location
  e. Participants expected
  f. Check-ins already made
  g. Button to go to Event Details Page (do not create this route yet)


## Libraries and technologies to use

- NextJS
- @tailwindcss/postcss
- Redux for global state management

If you think an additional library is necessary, 
stop and explain why before adding it. Wait for approval.

## Types

EVENT
* id: string;
* name: string;
* date: string;
* location: string;
* status: 'active' | 'closed' | 'cancelled';
* description: string;
* expected_count: number;
* checkin_count: number;
* error_count: number;
* entry_rate: number;

PARTICIPANTS
* id: string;
* event_id: string;
* name: string;
* type: 'vip' | 'normal';
* status: 'inside' | 'outside';
* checkin_count: number;

Feel free to create more types, as well as attributes for the types already descripted, but explain why.

## Third Party APIs

- BASE API = http://localhost:3001
- API to load events = GET BASE_URL/events.json
- API to load event details = GET BASE_URL/events/{event.id}.json
- API to get participants of an event = GET BASE_URL/participants?event_id=EVT-001


## Design

As a Event Panel, this panel should follow 
- Dark theme with a near-black navy background (#080c14). 
- Cards and panels use subtle white transparency (white/5, white/8) 
- with soft borders and a suble shadow instead of hard colors. Accent color is blue-500.
- Use rounded for all cards.

## Implementation details

- Use fetch to GET the data from the API
- Ensure that the page have 3 stats
  1. Loading - Data is being loaded
  2. Empty - No events to display
  3. Error - When the requisition catches an error
