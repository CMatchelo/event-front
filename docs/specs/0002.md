# Event Management
Create the event page, where it should have
1. Button Back to main 
2. Event Status
3. Event name
4. Event location
5. Event date
6. A board with 4 cards
  a. Number os participants expecteds (e.expected_count)
  b. Number of checkins (e.checking_count)
  c. Number of erros (e.error_count)
  d. Entry rate (e.entry_rate)
7. A graphic displaying the number of entries during the event where, checking checkin.success and checkin.action
  a. X axis - TIme by hour
  b. Y axis - Number of successfull entries
8. A graphic displaying the proportion between checkins that were successfull and errors

## Types

Checkin
* id: string;
* event_id: string;
* participant_id: string;
* timestamp: string;
* success: boolean;
* action: 'entry' | 'exit';
* error_reason: null|"already_checked_in"|"event_closed";

Create each type in a new file
Feel free to create more types, as well as attributes for the types already descripted, but explain why.

## Third Party APIs

- BASE API = http://localhost:3001
- API to load events = GET BASE_URL/events
- API to load event details = GET BASE_URL/events/{event.id}
- API to get participants of an event = GET BASE_URL/participants?event_id={checkin.id}
- API to get the checkins = GET /checkins?event_id={checkin.id}

## Design

As a Event Panel, this panel should follow 
- Dark theme with a near-black navy background (#080c14). 
- Cards and panels use subtle white transparency (white/5, white/8) 
- with soft borders and a suble shadow instead of hard colors. Accent color is blue-500.
- Use rounded for all cards.

## Implementation details

- Use fetch to GET the data from the API
- Ensure that the page have 3 stats
  1. Loading - Data is being loaded
  2. Empty - No events to display
  3. Error - When the requisition catches an error
