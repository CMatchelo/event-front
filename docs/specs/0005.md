# Event Management

## Libaries

- Jest 
- React Testing Library

## Tests

Use Jest and React Testing Library. Create a test for each of the following scenarios:

### Business Rules
- A VIP participant can check-in and check-out multiple times, as long as the event is not closed or cancelled
- A Normal participant can only check-in once. A second attempt should display an error message

### Component Behavior
- When clicking the check-in button on a participant row, it should call POST /checkins with the correct payload
- When loading the event page, the Loading state should be displayed
- When the API throws an error while loading the event, the Error state should be displayed
- When the API returns an empty participants array, the Empty state should be displayed

### Main Page
- When typing in the search field, only events matching the input should be displayed
